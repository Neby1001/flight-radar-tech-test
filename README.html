Flightradar24 Web FrontEnd FullStack Code Test

Overview

This zip-file contains a vite based vue application, and includes an express server that provides a few APIs. The styles are written using tailwindcss.

vue application

The vue application will render a table with a list of flights. The list is generated randomly when the node server starts.

Each flight has the following data:

  • id: a unique id for the flight
  • airline: the id of the airline that operates the flight
  • number: the flight number
  • status: whether the flight is live or scheduled
  • from/to: the id of the departure/arrival airport
  • std: the scheduled time of departure (date string)
  • sta: the sceduled time of arrival (date string)
  • latitude/longitude: the position of the aircraft
  • direction: where the aircraft is pointing to

express server

The express server provides the following APIs:

  • /api/flights: returns all the flights
  • /api/airports: returns all the airports
  • /api/airlines: returns all the airlines

The server code is in server.js.

Getting Started

You need to install the dependencies to initialize the project:

cd /path/to/fr24-web-frontend-codetest
npm install

To start the servers:

cd /path/to/fr24-web-frontend-codetest
npm run dev

The vite server will be listening on port 3000, while the express server on port 3001.

You can view the page by visiting http://localhost:3000.

The browser will automatically refresh when changing the vue code. You need to restart the node server if you change the APIs.
